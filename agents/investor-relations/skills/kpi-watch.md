# Skill: KPI Watch

Weekly scan of the metrics investors track. Flag any that moved materially this week. One-paragraph Slack summary — three bullets max, only the moves worth knowing about.

---

## When to invoke

Every Friday at 3pm ET (before Treasury's 4pm runway report, so this captures Monday–Thursday moves). On-demand from CFO requests ("anything moving this week?").

---

## Inputs

- This week's most recent FP&A KPI snapshot (if any — quarterly cadence, may not be fresh)
- This week's variance package (if mid-month and Controller has published)
- This week's Treasury runway report and cash projection
- This week's daily cash snapshots (for cash-trend recognition)
- Last 4 weeks' KPI-watch outputs from `~/finance-data/ir/kpi-watch/` — for trend recognition
- Configured KPI list (investor-tracked metrics) from `config.yaml` or `~/finance-data/ir/tracked-kpis.yaml`

---

## Process

### Step 1 — Read the tracked KPI list

From `~/finance-data/ir/tracked-kpis.yaml` (CFO-maintained), pull the canonical list of metrics this audience cares about. Default for Strand (PSP-shaped):

```yaml
tracked_kpis:
  - tpv_wtd
  - net_revenue_wtd
  - net_take_rate_rolling_30d
  - connect_take_rate_rolling_30d   # PSP-specific watch metric
  - operating_cash
  - runway_months
  - active_merchant_count
  - new_merchant_signings_wtd
  - fraud_loss_rate_rolling_7d
  - engineering_headcount
  - nrr_rolling_12mo
  - gross_churn_rate_rolling_12mo
```

### Step 2 — Pull the current value for each KPI

Read from the most recent canonical source for each:
- TPV / net revenue / take rates → operating model or Strand billing data
- Cash / runway → Treasury reports
- Headcount → payroll roster
- NRR / churn → variance package or CRM data

If a KPI's source is stale (no fresh data this week), label it "no fresh data" rather than reusing last week's value.

### Step 3 — Compare to prior week

For each KPI, compute:
- Week-over-week change
- 4-week trend (sparkline)
- Distance from any configured threshold

### Step 4 — Classify each KPI

Three buckets:

- **Material Moves** — the metric moved enough to be worth surfacing this week. "Enough" is configured per metric (default: >10% WoW for ratio metrics, >5% WoW for dollar metrics)
- **Stable** — no material movement this week
- **No fresh data** — couldn't get this week's value

### Step 5 — Write the report

Three sections:

1. **Headline** — one paragraph summarizing the material moves
2. **Material Moves** table
3. **Stable This Week** table
4. **Distance from Watch Thresholds** table
5. **Items Requiring Human Review** — typically empty; populated if a KPI moved into a critical zone or if a definitional question arose
6. **Methodology** + **Audit Trail**

### Step 6 — Post the Slack summary

Three bullets maximum. Material moves only. If nothing material moved this week, the post is one line: "📈 KPI Watch — Week N: No material moves this week. All thresholds nominal. [link]"

---

## Escalation thresholds

| Condition | Action |
|-----------|--------|
| Any tracked KPI moves >20% WoW | Surface in Material Moves with extra explanation context |
| Any tracked KPI moves >50% WoW | Halt, post to `#finance-alerts` for investigation before publishing the watch |
| Cash or runway crosses a configured threshold | Already covered by Treasury alerts; KPI Watch references the alert and notes it as a watch item |
| A KPI source is stale (>7 days) for two consecutive weeks | Flag in Items Requiring Human Review; ask CFO to refresh the source |
| A KPI has been Material Move 3 weeks in a row in the same direction | Flag as a structural trend, not a one-off — surface in Items Requiring Human Review with a "consider this as next month's update lead story" note |

---

## Anti-patterns to avoid

- **Don't surface noise.** If a metric moved within its normal volatility band, it's not Material — leave it in Stable.
- **Don't extrapolate.** The KPI Watch reports what happened; it doesn't project forward. Forward projection is the FP&A Analyst's job.
- **Don't compress to fewer than 3 bullets if 3 things moved.** Better to flag a small thing than miss it.
- **Don't expand beyond 3 bullets if only 3 things moved.** The CFO scans this in 20 seconds; padding it dilutes the signal.
- **Don't recompute metrics yourself.** Pull from the canonical sources (Treasury, FP&A, operating model). Discrepancies between sources are an Items Requiring Human Review entry, not yours to resolve.
- **Don't generate the Slack summary as a copy of the markdown.** The Slack summary is a fast-read; the markdown is the audit-quality record. Two different artifacts, same source.
