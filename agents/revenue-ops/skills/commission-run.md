# Skill: Commission Run

Monthly commission calculation from CRM closed-won data and comp-plan rules. Edge cases surface for human review; nothing auto-approved.

---

## When to invoke

Day 5 of new period, 9am ET. On-demand for off-cycle ("draft commissions for closed deals through this week").

---

## Inputs

- Closed-won deals from `hubspot` MCP for the period (close date in the prior month)
- Comp-plan rules from `~/finance-data/revops/comp-plans/<year>.yaml` (CFO-maintained)
- Prior period's commission run (for clawback context — were any deals canceled within the recovery window?)
- Each rep's quota and YTD attainment (from CRM custom fields or a separate file)
- Deal contract status (signed contract on file?)
- Configured thresholds

## Process

### Step 1 — Pull closed-won deals for the period

From HubSpot (or Salesforce), every deal where:
- Stage = Closed Won
- Close date falls in the target period

Include: rep, customer, ACV, contract term, signed contract on file (yes/no), any custom comp-relevant fields.

### Step 2 — Validate each deal

For every closed-won deal, check:
- Signed contract on file? — required for commission
- Deal value matches signed contract? — flag mismatches
- Custom terms (multi-year, prepay, custom rev-rec) noted?

Deals failing validation get held in "Edge Cases" — not excluded from commission run, but commission held pending resolution.

### Step 3 — Apply comp-plan rules per deal per rep

Read `comp-plans/<year>.yaml`. For each deal:
- Standard rate applies → compute base commission
- Accelerators (quota attainment >100%, new logo, strategic vertical, etc.) → apply if triggers met
- Multi-year terms → apply multi-year accelerator rule if defined
- Custom deal types (e.g., partner-sourced) → apply alternative rate

### Step 4 — Check for clawbacks from prior periods

Pull prior 60 days of deals (per the comp-plan recovery window). For any that were paid commissions but then canceled/refunded within the window:
- Clawback eligible
- Compute the clawback amount
- Apply as a negative line in this period's commission

### Step 5 — Surface edge cases

For every:
- Pending contract amendment
- Accelerator boundary case (where part of a deal earns the accelerator and part doesn't)
- Clawback application
- Missing-contract block
- Comp-plan ambiguity

Surface in "Edge Cases for Human Review." Each item describes the situation, the rule applied, and the proposed action. Human approves or adjusts before payroll inclusion.

### Step 6 — Compute totals

Per-rep totals + overall totals. Separate "Total commissions" from "Net for payroll inclusion (after edge-case holds)." The held amount becomes payable in a future run once the edge case resolves.

### Step 7 — Hand off to Payroll Reviewer

Write `~/finance-data/payroll/<year>-<month>/commission-input.md` with the per-rep payable amounts. Payroll Reviewer reads this for the next payroll cycle.

### Step 8 — Notify

Post summary to `#revops-ops` with the headline + edge-case count.

## Escalation thresholds

| Trigger | Channel | Action |
|---------|---------|--------|
| Signed contract missing on a closed-won deal | `#finance-alerts` | Block from commission run; flag for upload |
| Accelerator triggered for the first time this year for a rep | `#revops-ops` | Confirm rule interpretation with comp plan owner |
| Clawback >$5K on a single deal | `#revops-ops` | Surface separately; ensure rep was notified of cancellation |
| Total commission run >$X above plan (configurable) | `#revops-ops` AND `#finance-alerts` | Surface; likely a comp-plan threshold needs revisiting |
| Edge case where comp-plan rule is genuinely ambiguous | Halt | Surface to `#revops-ops`; do not draft a commission until human decides interpretation |

---

## Anti-patterns to avoid

- **Don't auto-approve any edge case.** Even "obvious" ones. Edge cases are where rep trust gets damaged when handled wrong.
- **Don't include unvalidated deals in totals.** Held items get their own column; don't blend.
- **Don't apply accelerators without verifying the trigger.** Quota attainment, especially at boundary cases (rep at 99.8% of YTD), needs human confirmation.
- **Don't recompute clawbacks silently.** If a prior period's commission has a clawback, list it explicitly as a negative line — never silently net it.
- **Don't write to CRM with computed values.** RevOps stays read-only on CRM.
