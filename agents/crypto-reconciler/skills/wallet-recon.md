# Skill: Wallet Reconciliation

Daily reconciliation of on-chain wallet balances against the accounting subledger. Multi-chain, multi-asset. On-chain truth is the source of truth.

---

## When to invoke

Daily at 8am ET (including weekends — crypto doesn't sleep). On-demand for any wallet that triggers an alert.

---

## Inputs

- Configured wallet list from `~/finance-data/crypto/wallets.yaml` — each wallet has: label, address, chain, custody type
- Configured asset list from `~/finance-data/crypto/assets.yaml` — each asset has: ticker, decimals, price oracle, accounting treatment
- Subledger balances from `bitwave` or `tres-finance` MCP — for each wallet, what does the subledger say the balance is?
- On-chain balances queried via:
  - The `gcp-blockchain` MCP for public-chain queries (Bitcoin, Ethereum, Solana, etc.)
  - The wallet-aware MCPs (`catalyx-wallet`, `enterprise-wallet`, `lightspark-sdk`) for custody-system reads
- Prior day's reconciliation snapshot
- Configured thresholds

## Process

### Step 1 — Pull configured wallets and assets

From `wallets.yaml` and `assets.yaml`. If either file is missing or stale (>30 days unmodified), surface a config-hygiene flag — these are the canonical truth about what's connected.

### Step 2 — For each wallet, query both sides

In parallel for performance:
- **On-chain query** via the appropriate MCP for the wallet's chain
- **Subledger query** for the same wallet's reported balance

If either query fails or times out, retry once. Second failure → halt that wallet's reconciliation; surface as a data-availability flag.

### Step 3 — Per-asset comparison

For each (wallet, asset) pair:
- On-chain balance
- Subledger balance
- Delta (signed, on-chain minus subledger)
- USD-equivalent of the delta (using configured price oracle)

If absolute delta < threshold ($1K or 0.5% of position, whichever larger): mark clean.
If absolute delta ≥ threshold: flag.

### Step 4 — For flagged wallets, surface the recent on-chain activity

Pull the trailing 7 days of on-chain transactions for this wallet. List the most recent ~10:
- Timestamp (UTC)
- Direction (in / out)
- Amount
- Counterparty address
- Subledger classification (if any) — pulled from subledger transaction log

The point is to make it obvious which recent transaction caused the discrepancy.

### Step 5 — USD-equivalent total

For each wallet, sum the per-asset USD values using the configured price oracle. Surface total holdings.

For stablecoins (USDC, USDT, DAI, etc.), validate the peg is holding. If a stablecoin is trading >1% off peg, flag separately — the USD-equivalent assumption breaks down.

### Step 6 — Multi-sig pending transactions

For each multi-sig wallet, check for pending transactions awaiting signers. Surface visibility (label, pending tx hash, description, signer status) — never approve.

### Step 7 — Cross-chain bridge detection

If a recent transaction looks like a bridge (locked on chain A, expected to appear on chain B), check both sides. If either side is unreconciled, flag the bridge transaction specifically — these are the easiest place for assets to "disappear."

### Step 8 — Write the report + notify

Save the markdown. Post the summary to `#crypto-ops`. Critical flags (unexplained large movements, smart-contract calls without classification) route to `#finance-alerts`.

## Escalation thresholds

| Trigger | Channel | Action |
|---------|---------|--------|
| Wallet delta ≥ threshold | `#crypto-ops` | Surface in daily summary with recent activity |
| Unexplained on-chain transaction (no subledger classification within 24h) | `#finance-alerts` | Surface with transaction detail; could be benign or could be a compromise indicator |
| Multi-sig pending transaction | `#crypto-ops` | Surface for visibility; read-only |
| Smart contract interaction without subledger classification | `#finance-alerts` | High-priority: could be a DeFi interaction OR a compromise |
| Stablecoin trading off-peg >1% | `#finance-alerts` | USD-equivalent assumption breaks |
| New asset detected in a connected wallet | `#crypto-ops` | Halt that wallet's reconciliation; require config update |
| MCP query failure / timeout twice in a row | `#finance-alerts` | Data availability issue |
| Total holdings move >10% in a single day | `#finance-alerts` | Surface with the contributing transactions |

## Anti-patterns to avoid

- **Don't aggregate USD across custody types in the headline.** Hot wallets, cold wallets, exchange custody, DeFi positions all have different risk profiles. The total is informative but the breakdown is essential.
- **Don't trust subledger over on-chain.** When they disagree, on-chain wins. The subledger needs fixing.
- **Don't assume a transaction is benign because the counterparty looks familiar.** Address-recognition shortcuts have been the start of multiple high-profile crypto losses.
- **Don't smooth bridge transactions.** A bridge has two on-chain events (one per chain); both need reconciliation. Treating it as one event loses visibility.
- **Don't include full wallet addresses in Slack summaries.** Use labels. Full addresses stay in local files only.
- **Don't reconcile stablecoins as exactly $1.** Note the actual price; the deviation matters when it's material.
- **Don't omit the methodology section.** Tax authorities, auditors, and future investigators will read this section to verify how a number was computed.
